# 竹筒围棋（Bamboo Cylinder Baduk）

一个左右首尾相接的竹筒围棋。默认使用兼顾全局与立体感的弧面视图，同时保留平面展开与完整 3D 竹筒；三种视图共享同一盘棋，随时可以切换。棋盘的行方向仍保留上下两条边界：同一行的第 0 列与最后一列互为邻点，因此没有传统棋盘的四个角。

## 在线游玩

**[立即开始竹筒围棋](https://bamboo-baduk.pystashell.workers.dev/)**

创建房间后，把页面生成的邀请链接发给朋友即可开始对局。第一位玩家执黑，第二位玩家执白，之后加入的玩家自动进入旁观席。

## 已实现

- 9×9、13×13、19×19 快捷尺寸，以及 5–25 路自定义尺寸
- 平面展开视图横向周期滑动，拖动后按整列吸附，可从任意切口观察全局
- 平面两侧明确标出首尾连通关系，左右边缘的列互为邻列
- 弧面模式把完整一圈弯向玩家，一次看清全局并保留 3D 深度；横向拖动时棋盘内容沿弧面首尾循环
- 平面、弧面与完整竹筒一键切换，棋局、落子、提子和点目状态完全共享
- 弧面支持周期平移与自动流动，完整竹筒支持自由旋转；两者都可使用滚轮或双指缩放
- 竹筒表面跨接缝连通、数气与提子
- 禁止自杀和位置超级劫
- 双方连续停一手后进入点目
- 点目阶段整组标记死子，可恢复对局处理争议
- 中国面积规则与日本数目规则
- 可配置贴目
- 在线房间：创建房间、复制邀请链接，第二位玩家自动执白，之后加入者自动旁观
- 服务端权威判定落子、提子、超级劫、停着、死子标记和点目；最终结果必须双方确认，任一方修改死子都会重置确认
- WebSocket 实时同步、刷新自动回座；主动退出才释放席位，房间 24 小时无活动后清理

## 本地运行

```powershell
$env:NODE_OPTIONS='--use-system-ca'
npm install
npm run dev
```

然后打开 `http://127.0.0.1:8787/`，创建房间并把邀请链接发给另一台设备。`npm run dev:online` 是同一个完整联机入口；只调试界面时可运行 `npm run dev:ui`，但该 Vite 入口不提供房间 API。联机服务使用 Cloudflare Worker 与 Durable Objects。

## 拓扑约定

对于大小为 `N` 的棋盘，点 `(row, col)` 的左右邻点为：

```text
(row, (col - 1 + N) mod N)
(row, (col + 1) mod N)
```

上下邻点只在 `row` 没有越过 `0` 与 `N - 1` 时存在。圆周使用 `N` 个不同的列，而不是把首尾两列渲染在同一条线上，因此 19 路仍然有完整的 361 个落子点。
